# 필요에 따라서 더 수정하겠습니다. 자주 생기는 문제가 있다면 말씀 해 주세요

0. 항상 코드를 수정하기 전, git pull origin master 명령어를 입력해
Remote 저장소와 동일하게 코드를 업데이트 해 주세요!

1. git push가 안되는 경우!
  A. git push가 일부는 작동하지만, 새로 추가한 파일들은 커밋되지 않을때 :
      → git add . 을 입력해주세요.
         새로 추가된 파일은 git이 track 하기 전 까지, 변경사항이 커밋에 반영되지 않습니다.

  B. git push가 아예 안되는 경우 :
      → Remote 저장소와 사용자의 로컬 저장소가 서로 다른 기록을 가지고 있는 경우 push 가 되지 않습니다 .
         최선의 방법은 코드를 변경하기 전 git pull origin master 명령어를 통해 Remote와 일치하게
         업데이트 한 후에, 코드를 수정하는 것 입니다.

         하지만 이미 커밋이 요단강을 건너간 경우
      →  git fetch origin master (리모트의 기록을 다운받아옴)
      →  git rebase origin/HEAD (리모트의 가장 최신 기록에 변경사항을 덮음)
         이 방법을 시도해서 고칠 수 있습니다.

         이 두가지가 모두 안되는 경우, 서로 다른 사람이 코드의 같은 부분을 다르게 고친 경우입니다.
         이 부분은 둘 중 어느 변경사항을 적용할지 선택하여 커밋을 수정해야만 합니다.

2. 수정을 잔뜩 해 놓았으나, 이전 상태로 돌아가고 싶을 때
   → git stash 를 입력해 주세요
     아직 커밋 하지 않은 변경사항이 모두 삭제되고, 가장 최근의 커밋으로 상태가 복구됩니다.
